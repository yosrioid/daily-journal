# Daily Journal v1

Release date: 2026-06-19

## Status

Daily Journal v1 is ready for personal MVP use.

This release covers the required v1 scope from `AI_CONTEXT.md`: Telegram
journal input, user-owned journal storage, daily lookup, delete latest, search,
basic mood and tag extraction, weekly reports, monthly reports, and automated
tests for core behavior.

This is not a polished consumer product. It is a maintainable backend MVP that
can be deployed for personal Telegram journaling after environment variables,
database migrations, and Telegram webhook configuration are set.

## Included

- FastAPI application with layered project structure.
- Environment-based configuration through `.env`.
- SQLAlchemy database models and Alembic migrations.
- Telegram webhook endpoint with secret-token validation.
- Telegram user auto-registration by `telegram_user_id`.
- Journal entry storage scoped to the requesting Telegram user.
- Outbound Telegram replies for saved journals and commands.
- Telegram commands:
  - `/start`
  - `/help`
  - `/today`
  - `/weekly`
  - `/monthly`
  - `/mood`
  - `/search keyword`
  - `/delete_last`
- Journal API for trusted internal clients.
- Report API for weekly and monthly reports.
- Rule-based mood and tag extraction.
- User ownership checks across journal and report operations.
- CI workflow running Ruff and Pytest on pull requests.
- Development workflow rules for phase branches, PR review comments, and CI.

## Data Ownership

User data is owned by Telegram account identity. The primary identity is
`telegram_user_id`; Telegram usernames are stored only as optional profile
metadata because they can change.

All user-owned journal and report access is scoped through the internal user
record resolved from Telegram identity.

## Security Notes

- `TELEGRAM_BOT_TOKEN`, `TELEGRAM_WEBHOOK_SECRET`, `INTERNAL_API_TOKEN`, and
  `AI_API_KEY` must come from environment variables.
- `.env` files must not be committed.
- Raw journal text and Telegram bot tokens must not be logged.
- Public journal access is not included.
- Journal API and report API are intended for trusted internal callers and
  require `X-Internal-Api-Token` plus `X-Telegram-User-Id`.

## Deployment Checklist

1. Create a Python 3.13 virtual environment.
2. Install dependencies with `python -m pip install -e ".[dev]"`.
3. Copy `.env.example` to `.env`.
4. Set `DATABASE_URL`.
5. Set `TELEGRAM_BOT_TOKEN`.
6. Set `TELEGRAM_WEBHOOK_SECRET`.
7. Set `INTERNAL_API_TOKEN` for internal API access.
8. Run `alembic upgrade head`.
9. Start the app with `uvicorn app.main:app`.
10. Configure the Telegram webhook to call `POST /telegram/webhook` with the
    configured secret token.

## Known Limits

- AI provider integration is optional and not required for v1.
- Mood and tag extraction are rule-based, not model-based.
- No web dashboard is included.
- No mobile app is included.
- No export to Markdown or PDF is included.
- No reminder notification system is included.
- No voice, image, or file-based journal input is included.
